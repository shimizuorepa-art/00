# 半自動ライティング運用

## 目的

AIエージェントが、キャラ会議、構成、設計、本文、文体、編集の流れで小説を継続制作する。

工程は本文を縛るためではなく、各担当が気持ちよく腕を出すために使う。

## 基本

- 旧15話はアーカイブ済み。骨子と材料として参照する。
- 新しい採用本文は `20_校了品/01_話別完成稿` に保存する。
- 各工程は5つのポジティブな仕事を持つ。
- 章骨子を作る段階で最新の人物設定を読み、その後はキャラ会議と章構成を主資料にする。
- 補佐くんは正式本文を黙って代筆せず、必要な担当サブエージェントへ発注する。

## 章の流れ

1. 章末人物更新で、前章の人間関係とキャラ変化を整理する。
2. キャラ会議で、2〜3人の欲しいもの、避けたいこと、噛み合わなさを出す。
3. 構成作家が、キャラ会議の火種から章構成メモを作る。
4. 各話ごとに、発注カード、設計メモ、本文初稿、文体調整、編集確認、完成稿で回す。
5. 章末に人物更新と章末レビューを作り、次章へ渡す。

## 通常回の成果物

| ファイル | 担当 |
|---|---|
| `00_発注カード.md` | 編集、構成作家 |
| `01_設計メモ.md` | 構成、原作、考証 |
| `02_本文初稿.md` | 脚本 |
| `03_文体調整.md` | 文体 |
| `04_編集確認.md` | 編集 |
| `05_完成稿.md` | 編集 |

## 差し込み担当

読者視点、整合性管理、プロデューサー、肯定レビュー、否定レビューは常設通過にしない。

使う場面:

- 読者が迷いそうな回
- 重要設定を増やす回
- 人物関係が大きく変わる回
- 編集が判断を迷う回
- 章末で全体を見たい回

## サブエージェント運用

- 発注は `00_運用管理/サブエージェント軽量発注テンプレート.md` を使う。
- 渡すのは、今回の仕事、受け取る材料、変えるもの、任せること、寝かせるもの、出力の6つを基本にする。
- 読む資料は、主要入力、補助資料、必要なら見る、に分ける。
- 寝かせるものは最大3つに絞る。
- 「禁止」「必ず」「漏れなく」を増やさず、担当が判断してよい余白を渡す。
- 役割が終わったサブエージェントは閉じる。
- 複数担当を使う時は、書き込み先を分ける。
- 補佐くんが仮稿を書く場合は、事前にオーナーへ確認し、`補佐くん仮稿` と明記する。
