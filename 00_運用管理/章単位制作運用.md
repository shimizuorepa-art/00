# 章単位制作運用

## 基本方針

オーナーは原則として章ごとに確認する。章内10話の構成、場面継続・転換、各話の役割、情報開示の配分は、補佐くん、構成作家、編集、必要に応じてプロデューサーが判断する。

1章は10話を基本単位とする。全体を何章にするかは未定のまま扱い、章が終わるごとに次章へ進むか、章数・焦点を見直す。

章内で1つの場面や会話を引っ張りすぎないため、章開始時に各話の開始点、終了点、切りどころを仮決めする。本文制作中に良い会話や情緒が伸びても、各話の役割を果たしたら原則そこで切り、続きは次話または章内の別ポイントへ回す。

10話構成は設計図であり檻ではない。ただし、10話から外す判断は各話制作中に場当たりで行わず、章構成メモまたは章末レビューで理由を残す。

## プロローグ運用

第1章に入る前に、必要に応じてプロローグを置く。プロローグは章内10話の基本単位に含めない。

プロローグの目的:

- 導入で読者を作中映画の情緒へ入れる。
- 「異世界なのか」と読める違和感またはジャンル混線を起こす。
- 劇中劇のカット後、撮影現場側へ戻り、主要登場人物を顔見せする。
- ここまで完了したら、第1章へ進む。

既存の第1話、第2話はプロローグ扱いとする。プロローグ完了までは、第1章10話構成を開始しない。

## 役割分担

- 補佐くん: オーナー意図を採用済み判断へ落とし、章制作の進行をつなぐ。
- 構成作家: 章内10話の配置、各話の役割、場面継続・転換、視点切替、引きの強弱を判断する。
- 編集: 各話の本文品質、合否、リテイク、正本反映を判断する。
- プロデューサー: 章数、章の大目的、オーナー判断が必要な分岐、制作優先順位を判断する。
- 原作: 構成メモを受けて、その回の原作案を書く。

## 章ごとの固定判断

章開始時に、オーナーが出す材料と制作側で決める材料を分ける。

オーナーが出す材料:

- この章で描きたいこと。
- この章で到達したい状態。
- 絶対に入れたい出来事、台詞、感情。
- やりすぎたくない要素。

補佐くん、構成作家、編集、必要に応じてプロデューサーが決める材料:

- 章の開始地点と終了地点。
- 章内10話の役割分担。
- 各話の開始点と終了点。
- どの場面を継続し、どこで転換するか。
- どの会話を次話へ回すか。
- どこを圧縮し、どこを見せ場にするか。
- 10話目の決着と次章への引き。

各話の基本判定:

- この話で何が変わるか。
- 誰の感情、判断、関係、情報が動くか。
- ラストで読者をどこへ引っ張るか。
- 役割を果たした後に残った会話や余韻は、次話へ送るか削る。

## 章内10話の標準役割

章構成メモでは、まず以下の標準役割に仮配置する。章の内容に合わせて入れ替えてよいが、各話に役割がない状態で本文制作へ入らない。

| 話数 | 標準役割 |
|---|---|
| 1話 | つかみ、異変、章目的の提示 |
| 2話 | 状況整理、主人公または視点人物の立ち位置 |
| 3話 | 最初の行動、軽い成功または失敗 |
| 4話 | 関係性、世界観、現場事情を深める |
| 5話 | 中盤の山、問題の拡大 |
| 6話 | 転換点、見えていた話の裏返り |
| 7話 | 追い込み、停滞や逃げ道の消失 |
| 8話 | 決断、覚悟、代償 |
| 9話 | クライマックス直前、最後の押し込み |
| 10話 | 決着、余韻、次章への引き |

## 章開始フロー

章を始める前に、構成作家は `章構成メモ.md` を作る。

章構成メモに必ず入れるもの:

- 章番号と仮タイトル。
- 章の大目的。
- 章の開始地点と終了地点。
- 章内10話の仮配置。
- 各話の役割。
- 各話の開始点、終了点、切りどころ。
- 継続予定の場面と、転換予定の場面。
- 絶対入れる場面、入れすぎ注意の要素。
- 主に動かす人物。
- 章内で出してよい情報。
- 章内でまだ隠す情報。
- 章末でオーナーに確認する論点。

## 各話開始フロー

各話の原作発注前に、構成作家は対象ラウンドへ `00_構成メモ.md` を作る。

`00_構成メモ.md` に必ず入れるもの:

- 章内位置。例: 第1章3/10話。
- 今回の話の役割。
- 前話からの継続か転換か。
- 場所、時間、視点を動かすか。
- 同じ場面を続ける場合、前話から何が変わるか。
- この話で何が変わったら切るか。
- この話に入れず、次話または別回へ送るもの。
- 今回のラストで次へ渡すもの。
- 今回出してよい情報。
- 今回まだ隠す情報。

その後、編集が `00_編集発注.md` を作り、原作へ発注する。

## 場面継続・転換の判断

基本ルールは「1話につき、必ず何かが1つ変わる。変わったら切る」。

同じ場面を続けてよい条件:

- 人物の位置、判断、関係、知っている情報のどれかが前進する。
- 前話のラストで残した引きを、本文行動で受けられる。
- 同じ小道具や場所を使っても、意味が変化している。
- 継続理由を `00_構成メモ.md` に一文で説明できる。

場面転換した方がよい条件:

- 同じ場所で3話以上続き、人物の判断が停滞している。
- 読者が新しい人物、場所、現実階層、制作側の事情を必要としている。
- 章内10話の中で、情緒回が続きすぎてウィットや現場の動きが薄くなっている。
- 次の回で時間、場所、視点を動かした方が、前話の余韻が強くなる。
- 今回の役割をすでに果たしているのに、会話や余韻だけで続いている。

## 章末フロー

10話分が揃ったら、構成作家と編集は `章末レビュー.md` を作る。

章末レビューに入れるもの:

- 章の大目的を達成したか。
- 各話の役割が重複しすぎていないか。
- 情報開示が早すぎる、遅すぎる箇所がないか。
- 次章へ持ち越す未解決事項。
- オーナーに確認する選択肢。

章末レビュー後、オーナーにまとめて確認を戻す。

## オーナー確認の原則

オーナーへ毎話確認を求めない。以下の場合だけ、章中でも相談してよい。

- 章の大目的が変わる。
- 新しい章数、主要人物の退場、重要設定の確定など、大方針に触れる。
- 既存の採用済み判断と矛盾する案が出た。
- AI側だけでは採用すべきでない倫理的、作品方針的な判断が出た。

それ以外は、章末でまとめて確認する。

オーナーの章ごとの口出しは、章の目的、到達点、絶対に入れたい要素、避けたい要素を中心に扱う。章内の細かな場面転換、各話の切りどころ、会話の圧縮は制作側で決める。
