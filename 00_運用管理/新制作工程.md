# 新制作工程

## 目的

本文をチェックリストの通過物にせず、キャラクターが物語を押し返してくる制作環境を作る。

旧15話で得た骨子、出来事、人物配置は参照する。ただし本文は新工程で書き直す。

## 基本方針

- 各工程の役割は5つのポジティブな仕事に絞る。
- 安全確認は裏側に置き、工程mdの正面には「何をよくするか」を書く。
- 章の骨子を作る段階でのみ、最新の人物深掘り正本と相関図をしっかり読む。
- 章骨子の後は、キャラ会議、章構成メモ、発注カードを主資料にして、キャラの声と行動を前へ出す。
- 章末ごとに、変わった人間関係とキャラ設定を更新する。
- サブエージェントへの発注は、`サブエージェント軽量発注テンプレート.md` に沿って短く渡す。

## 章開始フロー

1. `章末人物更新.md` を確認する。
2. 最新の人物深掘り正本と相関図を読む。
3. 次章のキャラ会議参加者を2〜3人選ぶ。
4. `キャラ会議.md` で、欲しいもの、避けたいこと、噛み合わなさ、関係の火種を出す。
5. 構成作家が `章構成メモ.md` へ章骨子をまとめる。
6. 編集が章骨子を見て、各話制作へ進める。

## 各話制作フロー

通常回は以下の6ファイルで回す。

| ファイル | 役割 |
|---|---|
| `00_発注カード.md` | この話で何が変わるかを短く渡す |
| `01_設計メモ.md` | 構成、原作、必要考証を一枚にまとめる |
| `02_本文初稿.md` | 初稿の時点で読める本文を書く |
| `03_文体調整.md` | 声、手元、視線、間を増幅する |
| `04_編集確認.md` | 良い核、詰まり、通す判断を残す |
| `05_完成稿.md` | 採用候補として整えた本文を置く |

重要回、設定追加回、迷った回だけ、読者視点、整合性管理、プロデューサー、肯定/否定レビューを追加する。

## サブエージェントへの渡し方

発注は、担当を縛るためではなく、担当が気持ちよく判断できる場所を作るために使う。

基本形は以下の6つだけにする。

1. 今回の仕事
2. 受け取る材料
3. 変えるもの
4. 任せること
5. 寝かせるもの
6. 出力

読む資料は、主要入力、補助資料、必要なら見る、に分ける。章骨子後の各話制作では、人物深掘り正本を毎回フルで読ませず、章構成メモ、キャラ会議、発注カードを主資料にする。

「禁止」「必ず」「漏れなく」を増やすより、「任せること」と「寝かせるもの」を明記する。寝かせるものは最大3つまで。

## キャラ会議の位置づけ

キャラ会議は章を決定しない。

キャラたちは、次章で自分が欲しいもの、避けたいもの、相手に言いたくないこと、噛み合わないところを勝手に話す。編集と構成作家は、そこから火種を拾って章骨子へ変換する。

参加者は各章2〜3人。基本は2人、三角の圧や現場側/役者側/原作者のズレをぶつけたい時だけ3人にする。

偏り防止のため、同じ組み合わせは最低3章あける。前章で変化した人物を1人、まだ会議で喋っていない人物を1人入れる。

## 章末更新

章が終わったら、`章末人物更新.md` を作る。

更新するもの:

- 章で変わった人間関係
- 章で変わったキャラ設定
- 次章へ持ち越す火種
- 人物深掘り正本へ反映する内容
- 次回キャラ会議の参加候補

人物深掘り正本へ反映するのは、編集確認後に採用済みになった変化だけにする。迷いは相談メモか章末人物更新の保留欄に置く。

## 安全弁

- 補佐くんは正式本文を黙って代筆しない。
- サブエージェントに発注する時は、主要入力、任せること、寝かせるもの、出力先を渡す。
- 合格していない本文は `20_校了品/01_話別完成稿` に入れない。
- 旧15話は骨子参照用であり、そのまま正本へ戻さない。
- 作品方針、主要人物の退場、重要設定の確定はオーナーへ戻す。
