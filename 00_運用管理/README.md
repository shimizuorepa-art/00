# 編集チーム運用メモ

一人AI会社として、一本の小説を複数の専門担当が別ルートで進めるための作業場。

## 基本ルール

- 各担当は自分の部署フォルダ内で作業する。
- 各担当は作業開始時に、自分のフォルダの `agent.md` を必ず読む。
- 編集は作業開始時に、`70_オーナー窓口\00_現在の指示.md` を必ず確認する。
- 作品の正本は `20_校了品` に置く。
- 他部署へ渡す要望や疑問は `30_引き継ぎ` に書く。
- 完成稿へ入れる前に `40_レビュー` でレビューする。
- 1回のライティングは `60_制作ラウンド` に保存し、本文3,000字程度までにする。
- 思いつき、没案、試作は `90_アーカイブ` へ退避する。

## 推奨ワークフロー

1. 原作が章ごとの狙い、プロット、展開候補を出す。
2. 考証が舞台、物品、制度、言葉遣いの裏取りをする。
3. 脚本が本文へ落とし込む。
4. 編集が肯定レビューと否定レビューをサブエージェントで行う。
5. 編集が両レビューを統合し、合格またはリテイクを判断する。
6. 必要に応じて原作、考証、脚本へ差し戻す。
7. 合意した稿だけを `20_校了品` に反映する。
8. 編集が次回ライティングの構想を原作へ発注する。

## 追加すると強い担当

- 文体: 文章の声、比喩、リズム、語彙の統一。
- 演出: 場面の見せ方、会話の間、映像的な段取り。
- 読者視点: 初見読者として、わかりにくさ、退屈さ、期待外れを指摘。
- continuity: 伏線、設定、時系列、人物感情の整合性管理。
- プロデューサー: 全体方針、締切、優先順位、どの案を採用するかの判断。
